# Requirement — Work Area scope

A Work Area may contain local tools, deployable services, libraries,
documentation, methodology, data, or mixed content. Repository governance owns
the declaration; session continuity may reference it but cannot redefine it.

## Primary claim

DSET supports repository-level scope or declared repository-relative Work Areas without assuming code, deployability, service, feature, module, or architecture type.

## Rationale

Monorepos and mixed repositories need bounded governance without forcing every governed folder into a software-service taxonomy.
