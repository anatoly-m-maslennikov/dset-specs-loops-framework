# Requirement — User Story framing

When useful, a Requirement may carry or link actor, desired capability, and
value framing. Independently enforceable acceptance criteria remain sibling
atomic Definitions.

## Primary claim

Actor, capability, and value framing is optional Requirement content or context rather than a routing axis or mandatory registered artifact name.

## Rationale

User-story wording can clarify a Requirement without creating a nested semantic hierarchy or combining independently enforceable acceptance claims.
