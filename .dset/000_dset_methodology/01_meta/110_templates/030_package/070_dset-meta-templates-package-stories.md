# {{title}} user stories

User Stories are optional. Add one only when a meaningful actor or stakeholder
wants a capability or outcome for a concrete value or purpose. A User Story is traceable,
but it is not an intake queue and does not replace normative Requirements.

## {{project_key}}-STORY{{id_layer}}-001 — User Story title

- **Actor or stakeholder:** Name the actor or stakeholder.
- **Desired capability or outcome:** State what they want.
- **Value or purpose:** State why it matters.
- **Linked Requirements:** Link the normative, verifiable WHAT.
- **Linked Scenarios:** Link observable behavior examples.

If no meaningful User Story exists, remove the example and leave `stories: []`
in `package.toml`; do not invent one.
