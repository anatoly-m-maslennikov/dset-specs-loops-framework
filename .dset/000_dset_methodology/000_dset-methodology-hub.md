# Installed DSET methodology

## Purpose

Own the accepted project-local methodology edition resolved by DSET workflows.
It contains governing rules, procedures, schemas, templates, and workflow
definitions.

## Boundaries

This area never contains applied project atoms, plans, evidence, or project
specifications. Skills resolve its unique identities only inside the target
repository's `.dset` control root.

## Start here

- `000_dset-project-hub.md` — cross-layer methodology surface.
- `000_dset-meta-hub.md` — specification and proof-definition rules.
- `000_dset-gov-hub.md` — artifact, authority, lifecycle, and compilation rules.
- `000_dset-tool-hub.md` — implementation and deterministic tool rules.
- `000_dset-skill-hub.md` — workflow gates, chaining, and handoffs.
- `000_dset-ops-hub.md` — delivery, release, and supportability rules.

Framework updates materialize an accepted source edition here; they never
overwrite applied project artifacts.
