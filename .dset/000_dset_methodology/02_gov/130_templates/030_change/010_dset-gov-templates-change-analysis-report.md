# Analysis report — {{title}}

## Question and boundary

State the question, intended use, subject, time boundary, and explicit
non-goals. Analysis informs authority; it never becomes accepted authority by
existing.

## Inputs and method

List evidence addresses, versions or observation windows, assumptions,
comparison or investigation method, limitations, and currentness.

## Findings

Separate observed facts, interpretation, uncertainty, and counter-evidence.
Do not turn a recommendation into an accepted Decision inside this carrier.

## Consequences

Link any separately emitted Decision, Question, Problem, Test, or Evaluation.
Record the trigger that requires this analysis to be refreshed.
