# Prototyping

**Rule ID:** `DSET-RULE-PROTOTYPING`

## Workflow

1. Read the governing requirements, separate proof plans, solution landscape, constraints, and candidate versions. Refuse an undefined decision question or threshold.
2. State one falsifiable hypothesis, time/effort bound, representative cases, hard constraints, success/failure thresholds, and disposal rule.
3. Keep disposable work under the active change's bounded proof area. Record source, exact version/commit, license, setup, data class, and external access before copying or adapting material.
4. Apply comparable accepted tests and evals to every candidate, including applicable boundary, failure, retry/recovery, concurrency, migration, diagnostics, and exit behavior.
5. Separate observed evidence from vendor claims and measure uncovered requirements, adapter/policy glue, persistence, operations, privacy/security, lock-in, maintenance, and replacement cost.
6. Conclude adopt, adapt, build, or defer. Link bounded evidence and create a decision record when material.
7. Delete or quarantine disposable runtime artifacts. Promotion requires accepted design and normal implementation proof.

## Stop conditions

Stop for a hard licensing, security, residency, platform, or architecture violation; an expired timebox; or insufficient comparable evidence. Never weaken thresholds for a preferred candidate.
