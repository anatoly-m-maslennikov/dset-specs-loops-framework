# Version Scope — DSET 0.4 self-hosted core

## Promise

Deliver one complete, recursively self-hosted DSET core vertical cut on this
repository: authority, compiled truth, implementation, proof, project health,
review reconciliation, session continuity, and release preparation agree at
the same revision.

## Included outcomes and capabilities

- Executable four-Type semantic model and independent MECE artifact-role model.
- Immutable atomic records, append-only lifecycle/absorption, priority, and
  deterministic conflict routing.
- Generated project health and external review reconciliation.
- Current project-local governance, schemas, templates, thin skills, and CLI
  recursively validate this repository.
- Required host, platform, dependency, hosted delivery, Test, Evaluation, and
  archive evidence for the bounded core release claim.

## Exclusions

- JavaScript/TypeScript applied enforcement.
- External adopter-readiness or Obsidian Your Harness pilot completion.
- Stable `1.0` compatibility guarantees.

## Exit criteria

- Every milestone in `DSET-PLAN-001` meets its linked owner conditions.
- `DSET-TASK-GOV-049`, `DSET-TASK-GOV-051`, `DSET-TASK-TOOL-043`,
  `DSET-TASK-GOV-044..045`, and `DSET-TASK-TOOL-046` are complete.
- Applicable host/platform/dependency/publisher proof tasks are complete or the
  scope is narrowed by a new accepted Decision.
- Fresh Tests, Evaluations, Verification, and an explicit Readiness Record pass
  for the exact candidate.

## Governing provenance

Compiled from `DSET-DECISION-OPS-004` and linked accepted DSET Decisions. A new
Decision is required to change this promise, its exclusions, or exit criteria.
