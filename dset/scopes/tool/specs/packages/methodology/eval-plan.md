# Methodology TOOL qualitative eval plan

Applicable qualitative and probabilistic proof is layer-owned. The representative scenario set is maintained by META; each row below owns its criterion and threshold.

| Eval ID | Criterion | Threshold |
|---|---|---|
| **DSET-EVAL-TOOL-001** | CLI diagnostic usefulness | A contributor can correct a malformed fixture from the stable code, path, and message without reading validator source |
| **DSET-EVALUATION-TOOL-003** | TypeScript profile portability | Independent reviewers apply the candidate to the owned pilot and a clean upstream fixture, distinguish language rules from product rules, preserve hard failures, treat baselines as bounded debt, and reach the same promote/revise/keep-candidate result from the evidence |
| **DSET-EVALUATION-TOOL-004** | Profile-role clarity | Reviewers distinguish the framework reference from OYOHA's applied instance, refuse reference-as-adopter-authority, classify concrete fields at their project owner, and locate the local materialization boundary without weakening blockers or promotion gates |

## Calibration and evidence

Use at least two independent reviewers for a baseline where interpretation matters. Record disagreements by case and criterion, and correct the earliest ambiguous owner rather than averaging incompatible results.
